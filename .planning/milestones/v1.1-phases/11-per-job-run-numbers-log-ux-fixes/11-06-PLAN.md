<objective>
Eliminate the UI-19 "error getting logs" race by moving the `job_runs` row insert from
the scheduler loop to the API handler thread for manual Run Now. The handler now (1)
validates CSRF, (2) looks up job, (3) INSERTs the row synchronously via
`insert_running_run`, (4) sends a new `SchedulerCmd::RunNowWithRunId { job_id, run_id }`
to the scheduler, (5) returns `HX-Refresh: true`. By the time the browser follows the
refresh and hits the run-detail URL, the row exists — no 404, no sseError flash.

**Legacy variant kept per RESEARCH Q1 RESOLVED:** `SchedulerCmd::RunNow { job_id }`
remains in the enum. Scheduled runs (cron tick path) still go through the scheduler-driven
`run_job` which inserts the row itself. Only manual "Run Now" clicks from the UI use the
new `RunNowWithRunId` variant. Both variants coexist. See Plan 11-06 Task 2 for the
coexistence rationale.

Purpose: Fix the sub-second race that painted "Unable to stream logs" for ~200ms after
every Run Now click. This is the headline UX bug Phase 11 closes.

            tracing::error!(target: "cronduit.web", error = %err, job_id, "run_now: lookup failed");
            return (StatusCode::INTERNAL_SERVER_ERROR, "Database error").into_response();

            return (StatusCode::INTERNAL_SERVER_ERROR, "Database error").into_response();
        }

    // Step 4 — CHANGED: dispatch the NEW variant carrying both ids.
    match state.scheduler_tx.send(SchedulerCmd::RunNowWithRunId { job_id, run_id }).await {

                job_id, run_id, job_name = %job.name,
                "run_now: row inserted + scheduler notified"
            );

        Err(_) => {
            let _ = queries::finalize_run(
                &state.pool, run_id, "error", None,
                tokio::time::Instant::now(),
                Some("scheduler shutting down"), None,
            ).await;
            return (StatusCode::SERVICE_UNAVAILABLE, "Scheduler unavailable").into_response();

    }

    // Step 5 — HX-Refresh + toast (unchanged).
    let event = HxEvent::new_with_data(

    ).expect("toast event serialization");
    let mut headers = axum::http::HeaderMap::new();

    (HxResponseTrigger::normal([event]), headers, StatusCode::OK).into_response()
}
