## Interaction States

### State 1 ‚Äî Run-history row, terminal run

- `#42` cell: `--cd-text-primary`, weight 400, size `--cd-text-base`. Bare numeral.
- `title="global id: 4711"` on the `<tr>` ‚Äî hover on any cell reveals the global id.
- Status badge carries terminal status (success / failed / timeout / stopped). No decoration on the `#N` cell itself.
- Stop column: empty `<td>` (no button, since `status != 'running'`). Table rhythm preserved.

### State 2 ‚Äî Run-history row, running run

- `#42` cell: **identical** visual to State 1. Bare numeral. No status-running tint on the `#N` cell (D-06 explicit ‚Äî status badge is the single source of truth).
- Status badge carries `running` (existing `--cd-status-running` blue).
- Stop column: compact Stop button (Phase 10 D-04) ‚Äî unchanged.
- Polling wrapper fires `hx-trigger="every 2s"` (existing behavior at `run_history.html:11`) ‚Äî unchanged.

### State 3 ‚Äî Run-detail header, running run

- Left flex item: `<h1>Run #42</h1>` immediately followed by `<span>(id 1234)</span>` ‚Äî same flex row, muted span.
- Right flex item: Stop button (Phase 10 D-03) ‚Äî unchanged.
- Layout: existing `flex items-center justify-between mb-6` container at `run_detail.html:16`. At 1280px+ operator viewports the full header fits without wrap. At narrower widths (‚â§ 640px) the muted suffix may wrap below ‚Äî acceptable; planner uses `flex-wrap` if necessary. No horizontal scroll on mobile.

### State 4 ‚Äî Run-detail header, terminal run

- Left flex item: same `Run #42 (id 1234)` layout as State 3.
- Right flex item: empty (Stop button gated by `is_running`).
- Breadcrumb tail shows `Run #42`.

### State 5 ‚Äî Run-detail, first paint of a running run (NEW ‚Äî D-08)

- Log-lines container (`#log-lines`) renders with `data-max-id="{{ last_id }}"` attribute present.
- If persisted lines exist: they render inline via the server-side include at `templates/partials/log_viewer.html`. No `Waiting for output...` placeholder. No skeleton flash.
- If zero persisted lines: the existing `Waiting for output...` placeholder renders (current behavior). HTMX SSE attaches immediately.
- Live SSE connection attaches via `hx-ext="sse" sse-connect="/events/runs/{id}/logs"`. New frames are appended via `sse-swap="log_line"` with `hx-swap="beforeend"`. Each new frame's `event.lastEventId` is compared to `container.dataset.maxId`; if `<=`, frame is dropped; otherwise appended and `dataset.maxId` is updated to the new id (D-09).
- Auto-scroll behavior (existing MutationObserver at `run_detail.html:100-109`) continues unchanged.

### State 6 ‚Äî Run-detail, live‚Üístatic transition (NEW ‚Äî D-10)

- Server emits `event: run_finished\ndata: {"run_id": N}\n\n` as the terminal SSE frame, then closes the stream.
- Client listens for `sse:run_finished` (or `sse:run_complete` ‚Äî planner picks the name) on `#log-lines`.
- On fire: `htmx.ajax('GET', '/partials/runs/{run_id}/logs', { target: '#log-container', swap: 'outerHTML' })` (existing pattern at `run_detail.html:118-121` ‚Äî keeps working).
- Any SSE frames still buffered client-side when the swap fires get dropped by D-09 dedupe because their `id <= new max` from the server-rendered static partial.
- **Zero visual animation.** Swap is instant. The operator perceives the badge flip from `running` ‚Üí terminal status and the `LIVE` pill disappearing ‚Äî that's it.
- Scroll position is preserved (only `#log-container`'s innerHTML changes, not the whole page ‚Äî D-11 explicit; do NOT reuse `HX-Refresh`).

### State 7 ‚Äî Run Now click-through race (UI-19, NEW behavior)

- **Before Phase 11:** clicking Run Now ‚Üí `HX-Refresh: true` ‚Üí page reloads ‚Üí `GET /jobs/{job_id}/runs/{run_id}` ‚Äî **but** the scheduler hadn't yet inserted the `job_runs` row, so the run-detail handler 404'd or rendered a half-state, causing the `htmx:sseError` branch to paint `Unable to stream logs. Refresh the page to retry.` for ~200ms before the real data arrived.
- **After Phase 11:** the `run_now` API handler (`src/web/handlers/api.rs:20-82`) synchronously inserts the `job_runs` row *before* returning the `HX-Refresh: true` response. By the time the browser follows the refresh and navigates to `/jobs/{job_id}/runs/{run_id}`, the row exists and the handler renders a normal running-run view (State 5).
- **Visual outcome:** the spurious flash never appears. The operator sees a clean transition from "Run queued: {job name}" toast ‚Üí page reload ‚Üí running-run detail page with `Waiting for output...` or first backfill lines, then live stream.

### State 8 ‚Äî Run-detail, terminal run, zero logs (existing, unchanged)

- Empty state renders verbatim from current template: `No log output captured for this run.` in `--cd-text-secondary` on `--cd-bg-surface`, centered, padding `--cd-space-6`.

- **New `#N` column:** leftmost, header empty or labeled `#` (planner discretion ‚Äî an empty `<th>` is consistent with the existing right-most "Stop" column header at `run_history.html:26` which is also empty). A single-character `#` header label is also acceptable if the planner wants it labeled.
- **Column min-width:** `<th>` + `<td>` for the `#N` column use `width:1%; white-space:nowrap;` so it collapses to content width and does NOT push other columns around. At 4-digit run numbers (`#9999`) the cell is ~40px wide at the base font size ‚Äî negligible.
- **At 5+ digit run numbers (> 9999):** cell gracefully widens because it's `width:1%` with no hard max. Unlikely in practice (a job running every minute for ~7 days hits 10k; the UI doesn't break, just uses a few more pixels).
- **Row-level `title` attribute:** attach to `<tr>`, not to `<td>`. This ensures hover anywhere on the row reveals the tooltip (better affordance than a tiny `#42` cell hit-target).
- **Cell padding:** `py-2 px-4` (same as existing cells) ‚Äî no override.

### Layout fits (viewport budget)

At a typical operator desktop viewport (1280+ px) with the existing `cd-container` max-width:

- Run-detail header row: `Run #42 (id 1234)` measures ‚âà180px at `--cd-text-xl` primary + `--cd-text-base` suffix. Stop button measures ‚âà64px. `justify-between` distributes the remainder. Total headroom: > 1000px free. No reflow.
- Run-history row with all columns at 1280+px viewport: 7 cells totaling ~900px under `overflow-x-auto`. New `#N` column adds ~50px at 4 digits. Still fits. At smaller viewports (< 900px) the existing `overflow-x-auto` at `run_history.html:17` handles overflow.

No media queries, no flex direction swaps, no column hides required.
