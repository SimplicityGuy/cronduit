- **Three rows for the semver family (`:X.Y.Z`, `:X.Y`, `:X`) instead of a single combined row.** ROADMAP says "five-tag contract" (counting the semver family as one) but the rendered table gives them separate rows because operators pinning `:1` vs `:1.0` vs `:1.0.1` have materially different reproducibility guarantees.
- **No new horizontal-rule between the new section and `## Architecture`.** README's existing H2-to-H2 convention (verified at Architecture → Configuration, Configuration → Monitoring, etc.) uses no horizontal rule between siblings. The pre-existing `---` at what was L76 remains as the Quickstart closer. Final count of `^---$` lines unchanged at 9.
- **Mermaid palette: three classDefs (`trigger`, `workflow`, `tag`).** Extends the existing Architecture diagram's two-tier `core`/`external` palette; three tiers needed here because trigger events, CI workflows, and published tags are semantically distinct.

## Threat Mitigation Evidence

**T-12.1-BLEED-01 (operator trust / `:main` surprises)** — verbatim mitigation wording now present in README.md:

> `:main` | A CI-built image of whatever commit is currently at the tip of `main` | Every push to `main`, multi-arch, built with the same toolchain as release images | Homelab operators who want the bleeding edge and can accept that `main` may be unstable

And reinforced in the "Picking a tag" section:

> `:main` is for operators who WANT the bleeding edge. It pulls unreviewed code from the tip of `main`; treat it the same way you would treat pulling from a branch of any open-source project. It is not recommended for any environment that values uptime.

**T-12.1-OPS-01 (`:latest` drift surprises)** — mitigated via explicit definition in the table:

> `:latest` | The most recent stable (non-rc) release -- currently `:1.0.1`, will advance to `:1.1.0` when it ships | Only on stable release tags (`vX.Y.Z` with no `-rc.N` suffix) | Operators who always want the newest stable. Never bleeds rc or main-branch builds in

And the "Picking a tag" steer toward pinning narrower (`:1.1`) for production.

**T-12.1-ASCII-01 (ASCII art violating CLAUDE.md)** — mitigated: diagram is a mermaid code fence, `grep -nE '[│├└─┬┴┼]' README.md` returns zero matches.
