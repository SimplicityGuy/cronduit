## OPS-10 Coverage

This plan delivers requirement **OPS-10** "new `:main` floating tag". The full OPS-10 contract is:

> Every push to `main` triggers a GitHub Actions workflow that builds a multi-arch (amd64+arm64) Docker image and publishes ONLY `:main` to `ghcr.io/simplicityguy/cronduit`.

All structural components are now in place:
- **Trigger:** `on: push: branches: [main]` ✓
- **Multi-arch:** `platforms: linux/amd64,linux/arm64` via `docker/build-push-action@v6` ✓
- **ONLY `:main`:** `tags: type=raw,value=main` (hardcoded — no branch-derived tags possible) ✓
- **Publishes to correct registry:** `ghcr.io/${{ env.IMAGE_NAME }}` with IMAGE_NAME lowercased ✓
- **Post-push verification:** `docker buildx imagetools inspect` asserts both archs are present ✓

**First real proof comes on the next push to `main` after this PR merges.** That is a MANUAL verification step (not in plan scope) and is covered by Plan 04's post-ship checkpoint and the `<verification>` block in this plan.
