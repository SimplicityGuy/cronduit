# Phase 12.1 Plan 01: ci.yml main-push GHCR cleanup Summary

**Deleted three GHCR-push steps from ci.yml image job (docker login + `just image-push` :latest + `just image-push` :sha-<7char>) that were overwriting `:latest` on every merge to main — replaced with a warning comment pointing at main-build.yml as the new owner of the `:main` tag.**

## Accomplishments

- Removed the structural mechanism that violated OPS-09 ("main-branch builds MUST NOT move `:latest`"): every merge to main previously ran `just image-push` (a full `docker buildx build --push`) against both `:latest` and `:sha-<7char>` GHCR tags.
- ci.yml image job is now PR-only by design — the only remaining step (`just image (PR -- build only)`) is gated on `github.event_name == 'pull_request'`, so push-to-main triggers a deliberate no-op.
- Added a forward-looking warning comment naming `main-build.yml` (Plan 02) as the new owner of the `:main` floating tag, with an explicit "DO NOT re-add a main-push step here" tied to the T-12.1-DRIFT-01 threat-register mitigation.
- Unblocks Plan 04 (`docker buildx imagetools create` retag) — without this deletion, the retag would be undone by the very next PR merge.

### Before/After excerpt

**Deleted block (verbatim, previously at L112-131):**
```yaml
      # main path: build and push to GHCR
      - name: docker login (main only)
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: just image-push (main -- latest)
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        env:
          REPO_OWNER: ${{ github.repository_owner }}
        run: just image-push "$(echo "${REPO_OWNER}" | tr '[:upper:]' '[:lower:]')/cronduit:latest"

      - name: just image-push (main -- sha)
        if: github.ref == 'refs/heads/main' && github.event_name == 'push'
        env:
          REPO_OWNER: ${{ github.repository_owner }}
        run: just image-push "$(echo "${REPO_OWNER}" | tr '[:upper:]' '[:lower:]')/cronduit:sha-${GITHUB_SHA:0:7}"
```

**Replacement comment block (now at L112-117):**
```yaml
      # main path: :main floating tag is owned by main-build.yml (OPS-10).
      # ci.yml's image job stays PR-only — verifies multi-arch BUILD works
      # without pushing. On push-to-main this job becomes a no-op because the
      # only remaining step (`just image (PR -- build only)` above) is gated on
      # `github.event_name == 'pull_request'`. DO NOT re-add a main-push step
      # here — that is what caused the pre-Phase-12.1 :latest divergence.
```

## Decisions Made

- **Follow plan verbatim.** The three-step deletion was already specified exactly in RESEARCH.md § "Root Cause Analysis" and PATTERNS.md § "ci.yml (L113-131)", so no interpretation needed.
- **Do not narrow `packages: write` to `packages: read`.** RESEARCH.md L674 lists this as an optional follow-up; leaving it alone keeps the diff minimal and reviewable. Threat-register disposition: `T-12.1-PERM-01 = accept` (permission is per-job scoped; no credentials exposed).
- **Do not rename `image` job.** Plan action block explicitly warns the name is in branch protection's required-checks list (RESEARCH.md Open Question #1 L710-714). Renaming would silently break branch protection.
