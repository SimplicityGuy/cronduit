1. Create `src/scheduler/docker_pull.rs` with:

   a. Error enum:
      ```rust
      #[derive(Debug)]
      pub enum PullError {
          /// Retryable: network timeout, connection refused, server 5xx
          Transient(String),
          /// Terminal: manifest unknown, unauthorized, invalid reference
          Terminal(String),
      }
      ```

   b. Error classification function:
      ```rust
      fn classify_pull_error(err: &bollard::errors::Error) -> PullError
      ```
      Classification rules:
      - If error string contains "unauthorized" or "authentication required" -> Terminal
      - If error string contains "manifest unknown" or "not found" -> Terminal
      - If error string contains "invalid reference" -> Terminal
      - Everything else (connection refused, timeout, 5xx, etc.) -> Transient

   c. Main pull function:
      ```rust
      pub async fn pull_image_with_retry(
          docker: &Docker,
          image: &str,
          max_attempts: u32,
      ) -> Result<Option<String>, PullError>
      ```
      Implementation:
      - Parse image into `from_image` (handle `name:tag` and `name` with implicit `latest`)
      - Loop for `max_attempts` (default 3):
        - Call `docker.create_image(Some(CreateImageOptions { from_image, .. }), None, None)`
        - Consume the entire stream via `while let Some(chunk) = stream.next().await`
        - On stream error: classify via `classify_pull_error`
          - If Terminal: return Err immediately (D-02)
          - If Transient and attempts remaining: log WARN with attempt number, reason, and next backoff duration (D-01):
            `tracing::warn!(target: "cronduit.docker.pull", image, attempt, max_attempts, reason = %err, backoff_secs, "pull attempt failed, retrying")`
          - Sleep for backoff: 1s, 2s, 4s (doubling)
        - On success: extract digest from `CreateImageInfo` stream. The last chunk with `id` field often contains the digest. Also check `status` field for "Digest: sha256:..." patterns.
        - On success: log INFO with only the resolved digest (D-03):
          `tracing::info!(target: "cronduit.docker.pull", image, digest = %digest, "image pulled")`
        - Return `Ok(Some(digest))` if digest extracted, `Ok(None)` if pull succeeded but no digest found.

   d. A helper function to check if image exists locally:
      ```rust
      pub async fn image_exists_locally(docker: &Docker, image: &str) -> Result<Option<String>, bollard::errors::Error>
      ```
      - Call `docker.inspect_image(image).await`
      - On Ok: extract digest from `ImageInspect.repo_digests` (first entry's sha256 portion)
      - On Err (404): return Ok(None) -- image not local
      - On Err (other): propagate error

   e. A combined entry point:
      ```rust
      pub async fn ensure_image(
          docker: &Docker,
          image: &str,
      ) -> Result<Option<String>, PullError>
      ```
      - First check `image_exists_locally`. If found, return the digest.
      - If not found, call `pull_image_with_retry(docker, image, 3)`.

1. Create `src/scheduler/docker_preflight.rs` with:

   a. Error enum (D-12: distinct error categories):
      ```rust
      #[derive(Debug)]
      pub enum PreflightError {
          /// Docker daemon is unreachable (socket error, permission denied)
          DockerUnavailable(String),
          /// Target container for container:<name> mode is not running
          NetworkTargetUnavailable(String),
          /// Named network does not exist
          NetworkNotFound(String),
      }

      impl PreflightError {
          /// Return structured error message for storage in job_runs.error_message
          pub fn to_error_message(&self) -> String {
              match self {
                  Self::DockerUnavailable(msg) => format!("docker_unavailable: {msg}"),
                  Self::NetworkTargetUnavailable(name) => format!("network_target_unavailable: {name}"),
                  Self::NetworkNotFound(name) => format!("network_not_found: {name}"),
              }
          }
      }
      ```

   b. Docker error classifier helper:
      ```rust
      fn classify_docker_error(err: bollard::errors::Error) -> PreflightError
      ```
      - If error is a connection/IO error -> DockerUnavailable
      - If error is 404 -> depends on context (caller determines)
      - Default -> DockerUnavailable with error message

   c. Main pre-flight function:
      ```rust
      pub async fn preflight_network(
          docker: &Docker,
          network_mode: &str,
      ) -> Result<(), PreflightError>
      ```
      Implementation:
      - If `network_mode` starts with `"container:"`:
        - Extract target name: `&network_mode["container:".len()..]`
        - Call `docker.inspect_container(target, None).await`
        - On error: classify -- 404 means container doesn't exist -> `NetworkTargetUnavailable(target)`; connection error -> `DockerUnavailable`
        - On success: check `response.state` -> `status` field. Use `ContainerStateStatusEnum::RUNNING` or match on the string representation "running"
        - If status is NOT running (stopped, paused, dead, created, etc.) -> return `Err(PreflightError::NetworkTargetUnavailable(target.to_string()))` (D-10)
        - If status IS running -> Ok(())
      - Else if `network_mode` is `"bridge"`, `"host"`, `"none"`, or `""`:
        - No pre-flight needed -> Ok(())
      - Else (named network):
        - Call `docker.inspect_network::<String>(network_mode, None).await` (D-11)
        - On error (404) -> `Err(PreflightError::NetworkNotFound(network_mode.to_string()))`
        - On error (connection) -> `Err(PreflightError::DockerUnavailable(...))`
        - On success -> Ok(())

2. Register in `src/scheduler/mod.rs`: add `pub mod docker_preflight;`

3. Add unit tests:

| Image registry -> Docker daemon | Image pull from potentially untrusted registries |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-07 | Tampering | Image pull from untrusted registry | accept | Operator chooses images in config file; no Cronduit-side registry allowlist in v1; documented in README |
| T-04-08 | Information Disclosure | Pre-flight reveals container existence | accept | Docker socket access already grants full visibility; pre-flight adds no new information disclosure surface |
| T-04-09 | Denial of Service | Retry loop on failing pulls | mitigate | Max 3 attempts with bounded backoff (1s, 2s, 4s = 7s max); terminal errors skip retry entirely |
