<objective>
Extend the existing `compose-smoke` GitHub Actions job (added in Phase 6 gap closure)
to run as a matrix over BOTH quickstart compose files and to assert that every example
job reaches `status=success` within 120 seconds of container boot. Closes the "quickstart
promise" contract for OPS-05 programmatically — every PR that touches Dockerfile,
examples/, or scheduler code now gates on a real end-to-end smoke test of both compose
variants.

This is the last gate before the human UAT walkthrough in Plan 08-05. Without CI
proof that the quickstart works, asking the user to walk through it manually is
asking them to debug CI-grade failures.

Purpose: The compose-smoke job in Phase 6 proves `/health` responds and both quickstart
jobs LOAD, but it never verifies they RUN successfully. Phase 8 closes that gap by
using the existing Run Now API to force execution, polling the run history endpoint,
and asserting terminal success within a 120-second budget. Running the assertions
against both compose files (default group_add + secure socket-proxy) keeps the dual-file
story real and not documentation-only.

Output: A single `compose-smoke` job parameterized by a `compose:` matrix axis. Each
axis runs one `compose up`, waits for /health, triggers all four example jobs via the
Run Now API, polls run history, asserts success, and tears down. On failure, dumps
logs and metrics for post-mortem.
</objective>

    BEFORE the "Dump compose logs on failure" step, insert:

    ```yaml
          - name: Trigger Run Now on every example job and assert success within 120s
            env:
              COMPOSE_FILE: ${{ matrix.compose }}
            run: |
              set -eu
              BASE="http://localhost:8080"
              JOBS="echo-timestamp http-healthcheck disk-usage hello-world"
              BUDGET_SECS=120
              POLL_INTERVAL=2

              # Fetch all jobs once and map name -> id (the Run Now API takes id, not name).
              jobs_json=$(curl -sSf "${BASE}/api/jobs")
              echo "---- /api/jobs body ----"
              echo "$jobs_json" | jq '.'

              for name in $JOBS; do
                id=$(echo "$jobs_json" | jq -r --arg n "$name" '.[] | select(.name == $n) | .id')
                if [ -z "$id" ] || [ "$id" = "null" ]; then
                  echo "ERROR: could not resolve job id for '$name' from /api/jobs"
                  exit 1
                fi
                echo "triggering Run Now for ${name} (id=${id})"
                curl -sSf -X POST "${BASE}/api/jobs/${id}/run" -o /dev/null || {
                  echo "ERROR: POST /api/jobs/${id}/run failed for ${name}"
                  exit 1
                }
              done

              echo "polling run history for every job (budget ${BUDGET_SECS}s, interval ${POLL_INTERVAL}s)"
              deadline=$(( $(date +%s) + BUDGET_SECS ))
              for name in $JOBS; do
                id=$(echo "$jobs_json" | jq -r --arg n "$name" '.[] | select(.name == $n) | .id')
                while :; do
                  now=$(date +%s)
                  if [ "$now" -ge "$deadline" ]; then
                    echo "ERROR: job '${name}' did not reach status=success within ${BUDGET_SECS}s"
                    echo "---- latest runs for ${name} ----"
                    curl -sSf "${BASE}/api/jobs/${id}/runs?limit=5" | jq '.'
                    exit 1
                  fi
                  latest=$(curl -sSf "${BASE}/api/jobs/${id}/runs?limit=1" | jq -r '.[0].status // empty')
                  case "$latest" in
                    success)
                      echo "job '${name}' reached status=success"
                      break
                      ;;
                    failed|timeout|cancelled)
                      echo "ERROR: job '${name}' reached terminal status=${latest} (expected success)"
                      curl -sSf "${BASE}/api/jobs/${id}/runs?limit=5" | jq '.'
                      exit 1
                      ;;
                    running|scheduled|"")
                      sleep "$POLL_INTERVAL"
                      ;;
                    *)
                      echo "WARN: unknown status='${latest}' for ${name}, continuing poll"
                      sleep "$POLL_INTERVAL"
                      ;;
                  esac
                done
              done

              echo "all four example jobs reached status=success within ${BUDGET_SECS}s"
    ```
