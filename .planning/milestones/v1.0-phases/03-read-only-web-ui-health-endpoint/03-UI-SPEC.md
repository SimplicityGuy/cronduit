# Phase 3 — UI Design Contract

> Visual and interaction contract for the Read-Only Web UI & Health Endpoint. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (server-rendered askama templates + Tailwind CSS standalone binary) |
| Preset | not applicable |
| Component library | none (hand-authored HTML components per `design/DESIGN_SYSTEM.md`) |
| Icon library | none (Cronduit logo mark SVG only; no general icon library in v1) |

## Spacing Scale

Declared values use a strict 4px grid. Only values from the standard set (4, 8, 16, 24, 32, 48, 64) are used:

| Token | Value | Usage |
|-------|-------|-------|
| `--cd-space-1` | 4px | Icon gaps, badge inline padding, tight element spacing |
| `--cd-space-2` | 8px | Compact element spacing, button inline padding, table cell vertical padding |
| `--cd-space-4` | 16px | Default element spacing, card inner padding (horizontal), form input padding, nav item spacing, filter bar gaps |
| `--cd-space-6` | 24px | Card/panel padding (all sides), section padding within pages |
| `--cd-space-8` | 32px | Layout gaps between major sections |
| `--cd-space-12` | 48px | Major section breaks (e.g., between filter bar and table), page horizontal margin on large viewports, touch target minimum height for "Run Now" button and nav links (accessibility) |
| `--cd-space-16` | 64px | Page-level vertical padding (top/bottom of main content area) |

Badge padding is `4px 8px` (`--cd-space-1` vertical, `--cd-space-2` horizontal). Touch target minimum height is 48px (`--cd-space-12`) for "Run Now" button and nav links (accessibility).

Source: `design/DESIGN_SYSTEM.md` section 4, adapted to strict standard set

## Typography

All text uses JetBrains Mono. No sans-serif or serif anywhere. This is the brand identity (DESIGN_SYSTEM.md section 3). Four font sizes, two font weights.

| Role | Token | Size | Weight | Line Height | Letter Spacing |
|------|-------|------|--------|-------------|----------------|
| Caption / Badge | `--cd-text-xs` | 0.65rem (10.4px) | 700 (bold) | 1.5 | 0.1em (uppercase) |
| Body / Secondary / Metadata | `--cd-text-base` | 0.9rem (14.4px) | 400 (regular) | 1.6 | 0 |
| Section heading | `--cd-text-lg` | 1.25rem (20px) | 700 (bold) | 1.4 | -0.02em |
| Page title | `--cd-text-xl` | 1.5rem (24px) | 700 (bold) | 1.3 | -0.02em |

Font weights: 400 (regular) for body text, metadata, and secondary content. 700 (bold) for headings, badges, and emphasis. No other weights.

Font fallback stack: `'JetBrains Mono', 'Fira Code', 'Source Code Pro', 'Cascadia Code', 'Consolas', monospace`

Source: `design/DESIGN_SYSTEM.md` section 3, consolidated from 6 to 4 sizes and 3 to 2 weights

## Color

All values use CSS custom properties that swap between dark and light mode via `data-theme` attribute on `<html>`. Dark mode is the default (`:root`). Light mode activates via `[data-theme="light"]` or `prefers-color-scheme: light` when no explicit theme is set.

### 60/30/10 Split

| Role | Dark Mode | Light Mode | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--cd-bg-primary` `#050508` | `#f8f8f6` | Page background, main content area |
| Secondary (30%) | `--cd-bg-surface` `#0a0d0b` | `#ffffff` | Cards, panels, table rows, nav bar, settings page sections |
| Accent (10%) | `--cd-green` `#34d399` | `#059669` | Reserved for: Run Now button, active nav indicator, focus rings, status badge "success/active" text, links, brand logo mark, table header sort indicator (active column) |

| Status | Foreground (dark) | Background tint (dark) | Semantic |
|--------|-------------------|------------------------|----------|
| Success / Active | `#34d399` | `rgba(52, 211, 153, 0.12)` | Successful run, healthy state |
| Running | `#60a5fa` | `rgba(96, 165, 250, 0.12)` | Job currently executing |
| Disabled / Warning | `#fbbf24` | `rgba(251, 191, 36, 0.12)` | Disabled jobs, timeout status |
| Error / Failed | `#f87171` | `rgba(248, 113, 113, 0.12)` | Failed run, error state |

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|
| `--cd-status-error` | `#f87171` | `#dc2626` | Error badges only. No destructive actions exist in Phase 3 (no delete, no cancel). |

### Additional Surface Colors

| Token | Dark Mode | Light Mode | Usage |
|-------|-----------|------------|-------|

Accent reserved for: Run Now button background, active nav link indicator, focus ring on inputs, `--cd-text-accent` links, brand logo mark color, sort-active column header indicator. Never used for general borders, backgrounds, or decorative elements.

## Component Inventory

### Primary Focal Point

The status badge column is the primary visual anchor on the Dashboard. It uses bold uppercase text, color-coded background tints, and high contrast to draw the operator's eye to job health at a glance.

### Pages (askama templates extending `base.html`)

| Page | Route | Template | HTMX Behavior |
|------|-------|----------|---------------|
| Dashboard | `GET /` | `dashboard.html` | Table partial polls `GET /partials/job-table` every 3s via `hx-trigger="every 3s"` |
| Job Detail | `GET /jobs/:id` | `job_detail.html` | Run history partial `GET /partials/run-history/:id?page=N` for pagination |
| Run Detail | `GET /jobs/:job_id/runs/:run_id` | `run_detail.html` | Log viewer partial `GET /partials/log-viewer/:run_id?offset=N` for "Load older lines" |
| Settings/Status | `GET /settings` | `settings.html` | Static render (no polling in Phase 3) |

### Partials (HTMX-swappable fragments)

| Partial | Route | Swap Target |
|---------|-------|-------------|
| Job table | `GET /partials/job-table?filter=&sort=&order=` | `#job-table-body` |
| Run history | `GET /partials/run-history/:id?page=N` | `#run-history` |
| Log viewer | `GET /partials/log-viewer/:run_id?offset=N` | Appended above existing log lines |

### API Endpoints

| Endpoint | Method | Purpose | CSRF Required |
|----------|--------|---------|---------------|
| `/api/jobs/:id/run` | POST | Trigger manual run via scheduler command channel | Yes |

## Interaction Contracts

### Dashboard Auto-Refresh

- HTMX polls `GET /partials/job-table` every 3 seconds via `hx-trigger="every 3s"`
- Server computes "next fire" as relative time (e.g., "in 4h 12m") -- no client-side JS timers
- Filter text input uses `hx-get` with `hx-trigger="keyup changed delay:300ms"` for debounced filtering
- Sort by clicking column headers: `hx-get` with `sort` and `order` query params
- Filter and sort state preserved in URL query params via `hx-push-url="true"`

### Run Now Flow

1. User clicks "Run Now" button (primary button on each job row or job detail page)
2. Button submits `POST /api/jobs/:id/run` with CSRF token as hidden field
3. Server validates CSRF, sends `SchedulerCmd::RunNow { job_id }` through `mpsc` channel
4. Response returns `204 No Content` with `HX-Trigger: showToast` header (carries job name in event detail)
5. Toast appears: "Run queued: {job_name}" -- auto-dismisses after 3s
6. Next 3s poll picks up the new running/completed row in the dashboard table

### Log Viewer Pagination

- Initial render shows most recent 500 log lines (tail of output)
- "Load older lines" secondary button at top of log viewer
- Button fires `hx-get="/partials/log-viewer/:run_id?offset=N"` and prepends older lines above existing content
- ANSI SGR codes parsed server-side into `<span class="ansi-*">` elements with CSS classes
- All log content HTML-escaped first, then ANSI transformation applied -- no `| safe` or `PreEscaped` on raw content

### Run History Pagination

- 25 runs per page (default)
- Previous/Next pagination links at bottom of run history table
- Links use `hx-get` with `page=N` query param, swap `#run-history` target

### CSRF Protection (Double-Submit Cookie)

- Server sets `HttpOnly; SameSite=Strict; Secure` cookie with random token on first page load
- Same token rendered as hidden `<input>` in forms containing state-changing actions
- Server validates cookie value matches form field value on POST
- No server-side session store required

### Dark/Light Mode Toggle

- Default: follow `prefers-color-scheme` media query
- Toggle button in nav (with `aria-label="Toggle dark/light mode"`) sets `data-theme="dark"` or `data-theme="light"` on `<html>`
- Choice persisted to `localStorage` key `cronduit-theme`
- JS snippet (~10 lines) runs on page load to restore preference before first paint (avoid flash)

## Accessibility Baseline

| Concern | Approach |
|---------|----------|
| Color contrast | All text/background pairs meet WCAG AA (4.5:1 minimum). Dark mode: `#d0ddd4` on `#050508` = 13.2:1. Light mode: `#1a1f1c` on `#f8f8f6` = 16.1:1. |
| Focus visibility | All interactive elements show `--cd-border-focus` ring (`--cd-green`) with `0 0 0 2px --cd-green-dim` box-shadow |
| Status communication | Status badges use text labels (not color alone): "SUCCESS", "FAILED", "RUNNING", "TIMEOUT", "ERROR" |
| Log viewer | Stderr lines distinguished by left-border AND background tint (not color alone) |
| Keyboard navigation | Tab order follows visual order. Run Now button, filter input, sort headers, pagination links all focusable. |
| Screen reader | Table uses `<th scope="col">` headers. Status badges use visible text (no icon-only indicators). |
| Theme toggle | Button declares `aria-label="Toggle dark/light mode"` for screen reader users. |
