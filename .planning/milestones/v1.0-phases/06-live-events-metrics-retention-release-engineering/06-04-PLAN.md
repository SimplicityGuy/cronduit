4. Complete `THREAT_MODEL.md` with the four models. Read the existing skeleton first and expand it. The four models per CONTEXT.md:

   **a. Docker Socket Model:**
   - Threat: Mounting /var/run/docker.sock gives Cronduit root-equivalent access to the host
   - Mitigations: Default loopback bind, read-only config mount, operator trust boundary (Cronduit is a tool for the operator, not a multi-tenant service), containers spawned with labels for orphan tracking, explicit auto_remove=false strategy
   - Residual risk: An operator misconfiguring a job's `image` or `command` can run arbitrary code with host Docker access

   **b. Untrusted Client Model:**
   - Threat: Network clients accessing the web UI or API without authentication
   - Mitigations: v1 ships unauthenticated by design (documented), default loopback bind, CSRF double-submit cookie on state-changing endpoints, SSE endpoints are read-only, /metrics is read-only
   - Residual risk: Anyone on the network can view job status and trigger manual runs; operators must use network controls or reverse proxy

   **c. Config Tamper Model:**
   - Threat: Attacker modifying the TOML config file to inject malicious jobs
   - Mitigations: Config mounted read-only in Docker, reload validates before applying, env-var interpolation uses process environment only (no external lookups), SecretString wraps sensitive fields
   - Residual risk: If host filesystem is compromised, config can be tampered (beyond Cronduit's control)

   **d. Malicious Image Model:**
   - Threat: Operator specifies a compromised Docker image in job config
   - Mitigations: Cronduit does NOT sandbox images (documented), operators must trust their image sources, `--security-opt no-new-privileges` and `--cap-drop=ALL` are recommended in documentation but not enforced in v1
   - Residual risk: A malicious image has full container capabilities and Docker socket access if mounted

   Use mermaid diagrams for trust boundary visualization (FOUND-11). Structure each model with: Threat, Attack Vector, Mitigations, Residual Risk, Recommendations.
