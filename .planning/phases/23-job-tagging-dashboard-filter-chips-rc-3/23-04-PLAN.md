## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CSS author → rendered chip | The CSS is static; the only operator-controlled data that flows through is the chip's text content (the tag name) which is HTML-escaped by askama 0.15 in Plan 23-05's template, NOT in the CSS layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-04-01 | Tampering | a malicious operator-supplied tag value breaks the chip rendering by escaping CSS | accept | Tags are validated against `^[a-z0-9][a-z0-9_-]{0,30}$` at config-load (P22 TAG-04). The CSS doesn't reference operator data — class names are static (`cd-tag-chip--active` etc.), tag values appear as TEXT children of `<a>` (escaped by askama). No CSS escape surface. |
| T-23-04-02 | Information Disclosure | hover/focus state leaks information about active chip ID | accept | Chip state is server-rendered + URL-shared. No client-side state. The `:hover` and `:focus-visible` rules are purely visual and reuse existing token vocabulary. |
| T-23-04-03 | Denial of Service | new CSS rules cause Tailwind compile failure | mitigate | `cargo build --quiet` is the build gate; if Tailwind rejects the new rules, build fails fast. Acceptance criterion: `cargo build` exits 0. |
| T-23-04-04 | Repudiation | new CSS introduces a new token quietly | mitigate | Acceptance criterion: `git diff assets/src/app.css | grep -E '^\+\s*--cd-'` returns 0 — no new token declarations. UI-SPEC § Tokens — Existing Reuse Verified is the explicit lock. |
| T-23-04-05 | Logic violation (a11y) | active state encoded by color alone (color-vision-deficient operator can't distinguish) | mitigate | Three-channel encoding per UI-SPEC § Color: `--cd-text-accent` border + `--cd-text-accent` label color + `font-weight: 700`. Acceptance criterion: the active rule grep contains all three properties. RESEARCH § Security Domain V11 (business logic; in this case a11y compliance). |

**Security gate:** No HIGH severity threats. Plan 23-04 is a pure CSS extension — no operator data flows through the CSS layer; the build gate catches malformed rules; the token lock prevents drift. The triple-channel active state encoding closes the only meaningful a11y risk.
