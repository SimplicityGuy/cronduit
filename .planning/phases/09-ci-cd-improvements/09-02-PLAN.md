```yaml
# .github/workflows/cleanup-images.yml
# Monthly GHCR image pruning. Keeps the package page navigable on a long-lived
# OSS project that publishes on every main-branch push plus every tagged release.
#
# Source of inspiration (matrix collapsed to single job — Cronduit ships ONE image):
#   https://github.com/SimplicityGuy/discogsography/blob/main/.github/workflows/cleanup-images.yml
#
# Retention policy:
#   - keep-n-tagged: 2       (keep the 2 most recent tagged releases regardless of age)
#   - older-than: 30 days    (delete anything older than 30 days beyond that)
#   - delete-untagged: true  (drop orphan blobs from superseded main builds)
#   - delete-partial-images: true

name: Cleanup Docker Images

on:
  workflow_dispatch:
  schedule:
    - cron: "0 0 15 * *"

concurrency:
  group: cleanup-images-${{ github.ref }}
  cancel-in-progress: false

permissions:
  contents: read

jobs:
  cleanup:
    name: prune ghcr.io/${{ github.repository_owner }}/cronduit
    runs-on: ubuntu-latest
    timeout-minutes: 30
    permissions:
      packages: write
      contents: read
    steps:
      - name: Cleanup Docker Images
        uses: dataaxiom/ghcr-cleanup-action@cd0cdb900b5dbf3a6f2cc869f0dbb0b8211f50c4 # v1.0.16
        with:
          delete-partial-images: true
          delete-untagged: true
          keep-n-tagged: 2
          older-than: 30 days
          token: ${{ secrets.GITHUB_TOKEN }}
          package: cronduit
          owner: ${{ github.repository_owner }}
```

Key structural differences from the discogsography source (intentional):
1. **No `list-sub-projects` upstream job.** Discogsography has `uses: ./.github/workflows/list-sub-projects.yml` because it publishes multiple sub-project images (`dashboard`, `discovery`, `extractor`, etc.). Cronduit publishes exactly ONE image. Drop that file reference entirely.
2. **No matrix strategy.** Discogsography has `strategy.matrix.package: ${{ fromJson(needs.list-sub-projects.outputs.simple-matrix) }}`. Remove the entire `strategy:` block.
3. **No `needs: [list-sub-projects]`.** Single job, no dependencies.
4. **`package: cronduit`** (literal string), not `${{ matrix.package }}`.
5. **`cancel-in-progress: false`** (match discogsography) — a cleanup in progress must not be cancelled by a new dispatch.
6. **Top-level `permissions: { contents: read }`** is added (discogsography relies on default); Cronduit's CONTEXT.md mandates explicit top-level permissions.
7. **Step name is plain text** (`Cleanup Docker Images`, no emoji) because the emoji exception in CONTEXT.md is scoped ONLY to `scripts/update-project.sh`.

Do NOT add any other `uses:` lines (no checkout — this workflow does not need the repo contents).
