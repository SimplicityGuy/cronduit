<objective>
Rebase the cronduit runtime image from `gcr.io/distroless/static-debian12:nonroot` to
`alpine:3` so non-Docker example jobs (`command=` / `script=`) can execute the busybox
tooling they need (`date`, `wget`, `du`, `df`, `/bin/sh`). Simultaneously rewrite
`examples/cronduit.toml` so the quickstart ships FOUR jobs covering every execution
type instead of the current two (which are both broken on the distroless runtime).

This plan closes blockers #1 (echo-timestamp ENOENT) and the "need sustained RUNNING
state" dependency for Plan 07-05's browser UAT (blocker #3) — `http-healthcheck` and
`disk-usage` naturally stay in RUNNING for several seconds each, enough to observe the
HTMX polling transition.

Purpose: The distroless rebase was a conscious Phase 1 decision for minimal attack
surface, but it's incompatible with the project's design contract "cronduit runs
command/script/docker jobs reliably and makes their state observable". If the
quickstart demo jobs cannot execute, the product promise is broken. Phase 8 walks
the distroless posture back to alpine — still non-root, still minimal, now actually
runnable — and records the rationale in the Dockerfile header for future-us.

Output: An alpine-based runtime image that boots as UID 1000 with a writable /data
volume, and a four-job quickstart config that demonstrates command, script, and
docker execution end-to-end.
</objective>

    ```toml
    # ------------------------------------------------------------
    # Quickstart jobs
    # ------------------------------------------------------------
    # Four jobs covering every execution type so a new operator can see
    # commands, scripts, and Docker containers run out of the box:
    #
    #   1. echo-timestamp   -- command   (every 1 min) instant heartbeat
    #   2. http-healthcheck -- command   (every 5 min) realistic uptime canary
    #   3. disk-usage       -- script    (every 15 min) /data volume inspector
    #   4. hello-world      -- docker    (every 5 min) ephemeral container demo
    #
    # All four run against the alpine:3 cronduit runtime image shipped in
    # Phase 8. The command/script jobs rely on busybox applets (date, wget,
    # du, df, /bin/sh); the docker job requires the host Docker socket mount
    # in docker-compose.yml (or the docker-socket-proxy sidecar in
    # docker-compose.secure.yml).

    # 1. Instant heartbeat. Runs every minute so you see it in the dashboard
    #    within ~60s of `docker compose up`.
    [[jobs]]
    name = "echo-timestamp"
    schedule = "*/1 * * * *"
    command = "date '+%Y-%m-%d %H:%M:%S -- Cronduit is running!'"

    # 2. Realistic uptime canary. Hits https://www.google.com using busybox
    #    wget, captures the response headers into cronduit's stdout log via
    #    `2>&1 | head -10`. Validates DNS + egress + TLS from inside the
    #    container. Swap the URL for your own uptime target when adopting.
    [[jobs]]
    name = "http-healthcheck"
    schedule = "*/5 * * * *"
    command = "sh -c \"wget -q -S --spider https://www.google.com 2>&1 | head -10\""

    # 3. Script-type job demonstrating the `script = """..."""` path and the
    #    /data named volume. Prints disk usage if /data is mounted, or a
    #    friendly message otherwise. Shebang resolves via alpine's /bin/sh.
    [[jobs]]
    name = "disk-usage"
    schedule = "*/15 * * * *"
    script = """#!/bin/sh
    du -sh /data 2>/dev/null || echo "/data not mounted"
    df -h /data 2>/dev/null || true
    """

    # 4. Ephemeral Docker container demo. Pulls hello-world:latest and runs it;
    #    `delete = true` removes the container on completion. Requires the
    #    Docker socket mount (docker-compose.yml) or the socket-proxy sidecar
    #    (docker-compose.secure.yml).
    [[jobs]]
    name = "hello-world"
    schedule = "*/5 * * * *"
    image = "hello-world:latest"
    delete = true
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host -> container filesystem | Named volume mount `cronduit-data:/data` crosses this boundary; new UID 1000 ownership must not break existing Phase 6 quickstart assumptions |
| container -> host network | Busybox wget to https://www.google.com crosses egress boundary (new) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Elevation of Privilege | alpine:3 runtime package surface | mitigate | Use `apk add --no-cache ca-certificates tzdata` only (2 packages on top of base alpine). Do not install bash, sudo, shadow, or any package that enables local privilege escalation. Document in Dockerfile header comment. |
| T-08-02 | Tampering | /data volume ownership on upgrade | accept | Operators upgrading from the Phase 6 distroless image (UID 65532) to Phase 8 alpine (UID 1000) will hit a permission error on their existing named volume. Release notes document the one-line `chown` remediation (handled in deferred Phase 8 scope per CONTEXT.md). Low blast radius — fresh installs are unaffected. |
| T-08-03 | Information Disclosure | http-healthcheck leaks target URL to logs | accept | `https://www.google.com` is a global public endpoint. No PII, no internal infrastructure exposure. Operators changing this URL to an internal endpoint do so consciously. |
| T-08-04 | Denial of Service | Four jobs in the default quickstart multiply scheduler load | accept | Schedule cadence (1/5/15/5 min) is well under any realistic scheduler-loop budget. Four jobs is still a toy config; production users run dozens. |
</threat_model>
