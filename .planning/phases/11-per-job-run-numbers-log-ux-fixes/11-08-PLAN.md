<objective>
SSE frame `id:` line emission (UI-18, UI-20): when the SSE handler receives a `LogLine`
with `id: Some(n)`, emit `id: {n}\nevent: log_line\ndata: ...\n\n` via
`axum::response::sse::Event::default().event("log_line").id(n.to_string()).data(html)`.
HTMX's SSE extension surfaces this as `event.lastEventId` on the `sse:log_line` DOM
event, which the client-side dedupe handler (Plan 11-11) will compare to `data-max-id`.

Purpose: The client dedupe contract (D-09) requires the server to publish the id on
every frame. This plan is the server-side half of the contract.
