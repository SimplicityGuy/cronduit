Open `.github/workflows/ci.yml`. At the end of the `jobs:` block (AFTER the existing `image:` / `build:` job, at the same two-space indentation level), append a new `compose-smoke` job.

**Critical pre-decided image strategy (do NOT deviate):**

The committed `examples/docker-compose.yml` keeps its `image: ghcr.io/simplicityguy/cronduit:latest` reference so a stranger following the README gets the published image. On CI however, `:latest` on a PR build is the PREVIOUS main build, which does not contain the PR's code — so a naive `docker compose up` would run stale code and the smoke test would not actually validate the PR.

To solve this without mutating the committed file, the smoke job:

1. Builds cronduit as `cronduit:ci` locally via `docker/build-push-action@v6` with `load: true` (loads into the runner's local Docker daemon, does not push).
2. Uses `sed` to rewrite `ghcr.io/simplicityguy/cronduit:latest` → `cronduit:ci` in `examples/docker-compose.yml` in the runner's ephemeral workspace ONLY (the rewrite is never committed, never pushed).
3. Runs `docker compose up -d` against the rewritten file, asserting `/health` and the dashboard.
4. Tears down on exit.

This way the smoke test exercises the PR's code. The committed compose file remains pointed at the published image for end-user quickstart, unchanged.

Insert this exact YAML block at the end of `.github/workflows/ci.yml` (after the last line of the final existing job, preserving existing two-space job-level indentation):

```yaml

  compose-smoke:
    name: quickstart compose smoke (examples/docker-compose.yml)
    runs-on: ubuntu-latest
    # GAP-3.4 (06-07-PLAN.md): bring up the quickstart compose file the stranger
    # following the README would use, and assert /health returns 200. Builds a local
    # cronduit:ci image from the PR checkout (NOT stale ghcr :latest) and rewrites
    # the committed compose file in-place in the runner's ephemeral workspace so
    # the smoke test exercises the PR's code, not the previous main build.
    steps:
      - uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build local cronduit:ci image from PR checkout
        uses: docker/build-push-action@v6
        with:
          context: .
          file: Dockerfile
          platforms: linux/amd64
          push: false
          load: true
          tags: cronduit:ci
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Rewrite compose to use locally-built cronduit:ci image
        run: |
          # The committed compose file references ghcr.io/simplicityguy/cronduit:latest
          # so end users pulling from README get the published image. On CI we swap it
          # for the locally-built PR image so the smoke test validates PR code, not
          # stale main. This rewrite happens in the runner's ephemeral workspace only
          # and is NEVER committed.
          sed -i 's|ghcr.io/simplicityguy/cronduit:latest|cronduit:ci|g' examples/docker-compose.yml
          grep -q 'image: cronduit:ci' examples/docker-compose.yml || {
            echo "ERROR: sed rewrite did not produce 'image: cronduit:ci' in compose file"
            exit 1
          }

      - name: docker compose up -d
        working-directory: examples
        run: docker compose -f docker-compose.yml up -d

      - name: Wait for /health (max 30s)
        run: |
          set -eu
          for i in $(seq 1 30); do
            if curl -sSf http://localhost:8080/health >/tmp/health.json 2>/dev/null; then
              echo "health responded after ${i}s"
              cat /tmp/health.json
              exit 0
            fi
            sleep 1
          done
          echo "ERROR: /health never responded after 30s"
          docker compose -f examples/docker-compose.yml logs
          exit 1

      - name: Assert /health body contains status:ok
        run: |
          set -eu
          body=$(curl -sSf http://localhost:8080/health)
          echo "health body: $body"
          echo "$body" | grep -q '"status":"ok"' || {
            echo "ERROR: /health body missing status:ok"
            exit 1
          }

      - name: Assert dashboard lists both quickstart jobs
        run: |
          set -eu
          dash=$(curl -sSf http://localhost:8080/)
          echo "$dash" | grep -q 'echo-timestamp' || {
            echo "ERROR: dashboard missing echo-timestamp job"
            echo "$dash"
            exit 1
          }
          echo "$dash" | grep -q 'hello-world' || {
            echo "ERROR: dashboard missing hello-world job"
            echo "$dash"
            exit 1
          }

      - name: Dump compose logs on failure
        if: failure()
        run: docker compose -f examples/docker-compose.yml logs

      - name: Tear down compose stack
        if: always()
        working-directory: examples
```
