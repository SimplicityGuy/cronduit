1. Create `.github/workflows/release.yml`:

   NOTE: The release workflow uses `docker/build-push-action@v6` directly rather than calling a justfile recipe. This is intentional -- the GitHub Action provides native GHA cache integration (`type=gha`), platform matrix support, and OCI label injection that are not available through a justfile wrapper. The justfile `image` recipe remains for local builds only.

   ```yaml
   name: Release

   on:
     push:
       tags: ['v*']

   permissions:
     contents: write
     packages: write

   env:
     REGISTRY: ghcr.io
     IMAGE_NAME: ${{ github.repository }}

   jobs:
     release:
       runs-on: ubuntu-latest
       steps:
         - name: Checkout
           uses: actions/checkout@v4
           with:
             fetch-depth: 0  # Full history for git-cliff

         - name: Extract version from tag
           id: version
           run: |
             TAG="${GITHUB_REF#refs/tags/v}"
             echo "version=$TAG" >> "$GITHUB_OUTPUT"
             echo "major=$(echo $TAG | cut -d. -f1)" >> "$GITHUB_OUTPUT"
             echo "minor=$(echo $TAG | cut -d. -f1-2)" >> "$GITHUB_OUTPUT"

         - name: Generate changelog
           uses: orhun/git-cliff-action@v4
           id: changelog
           with:
             config: cliff.toml
             args: --latest --strip header
           env:
             OUTPUT: CHANGELOG.md

         - name: Set up Docker Buildx
           uses: docker/setup-buildx-action@v3

         - name: Log in to GHCR
           uses: docker/login-action@v3
           with:
             registry: ${{ env.REGISTRY }}
             username: ${{ github.actor }}
             password: ${{ secrets.GITHUB_TOKEN }}

         - name: Build and push multi-arch image
           uses: docker/build-push-action@v6
           with:
             context: .
             platforms: linux/amd64,linux/arm64
             push: true
             tags: |
               ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.version.outputs.version }}
               ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.version.outputs.minor }}
               ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.version.outputs.major }}
               ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
             labels: |
               org.opencontainers.image.source=https://github.com/${{ github.repository }}
               org.opencontainers.image.version=${{ steps.version.outputs.version }}
               org.opencontainers.image.revision=${{ github.sha }}
               org.opencontainers.image.description=Self-hosted Docker-native cron scheduler with a web UI
               org.opencontainers.image.licenses=MIT OR Apache-2.0
             cache-from: type=gha
             cache-to: type=gha,mode=max

         - name: Create GitHub Release
           uses: softprops/action-gh-release@v2
           with:
             body: ${{ steps.changelog.outputs.content }}
             draft: false
             prerelease: ${{ contains(steps.version.outputs.version, '-') }}
   ```

   This triggers on `v*` tags (D-14), produces 4 image tags (version, minor, major, latest per D-14), uses git-cliff for changelog (D-16), and publishes to GHCR. First release will be `v0.1.0` (D-15).

2. Create `cliff.toml` for git-cliff configuration:

   ```toml
   # git-cliff configuration for Cronduit changelog generation.
   # Used by .github/workflows/release.yml via orhun/git-cliff-action.

   [changelog]
   header = """
   # Changelog\n
   All notable changes to Cronduit are documented here.\n
   """
   body = """
   {% if version %}\
       ## [{{ version | trim_start_matches(pat="v") }}] - {{ timestamp | date(format="%Y-%m-%d") }}
   {% else %}\
       ## [unreleased]
   {% endif %}\
   {% for group, commits in commits | group_by(attribute="group") %}
       ### {{ group | striptags | trim | upper_first }}
       {% for commit in commits %}
           - {% if commit.scope %}*({{ commit.scope }})*: {% endif %}\
               {{ commit.message | upper_first }}\
       {% endfor %}
   {% endfor %}\n
   """
   trim = true

   [git]
   conventional_commits = true
   filter_unconventional = true
   split_commits = false
   commit_parsers = [
       { message = "^feat", group = "Features" },
       { message = "^fix", group = "Bug Fixes" },
       { message = "^refactor", group = "Refactor" },
       { message = "^perf", group = "Performance" },
       { message = "^test", group = "Testing" },
       { message = "^docs", group = "Documentation" },
       { message = "^ci", group = "CI/CD" },
       { message = "^chore", skip = true },
       { message = "^style", skip = true },
   ]
   filter_commits = false
   tag_pattern = "v[0-9].*"
   sort_commits = "oldest"
   ```

3. Add OCI labels to the `Dockerfile`. Read the existing Dockerfile first. Add these `LABEL` instructions in the final stage (the runtime stage, after the `FROM` for the distroless/runtime image):

   ```dockerfile
   LABEL org.opencontainers.image.source="https://github.com/SimplicityGuy/cronduit"
   LABEL org.opencontainers.image.description="Self-hosted Docker-native cron scheduler with a web UI"
   LABEL org.opencontainers.image.licenses="MIT OR Apache-2.0"
   ```

   These are static labels. The dynamic labels (version, revision) are passed via `--label` in the GitHub Actions workflow's `docker/build-push-action` step.

4. Add a `release` recipe to the `justfile` for local release tagging convenience:

   ```
   # Tag and push a release. Usage: just release 0.1.0
   release version:
       @echo "Creating release v{{version}}..."
       git tag -a "v{{version}}" -m "Release v{{version}}"
       git push origin "v{{version}}"
       @echo "Release v{{version}} tagged and pushed. CI will build and publish."
   ```

   Place this in the meta section of the justfile. Note: this recipe is a convenience wrapper for tagging -- the actual image build and push happens in CI via `docker/build-push-action@v6`, not through justfile.
