<objective>
Implement the Prometheus /metrics endpoint with four metric families instrumented at the scheduler run lifecycle.

Purpose: Operators need observability into job execution patterns for alerting and dashboarding (OPS-02). The metrics facade decouples instrumentation from the exporter, and the closed-enum failure reasons prevent cardinality explosion.

Output: A `GET /metrics` endpoint returning Prometheus text format, instrumentation in the scheduler run finalization path, a `FailureReason` enum, and an `examples/prometheus.yml` scrape config.
</objective>

   // After computing status_str and duration:

   // Record run completion (D-07: include job label)
   counter!("cronduit_runs_total", "job" => job.name.clone(), "status" => status_str.to_string()).increment(1);

   // Record duration histogram (D-07: include job label)

2. Implement `classify_failure_reason()` in `src/scheduler/run.rs` that maps the run outcome to the FailureReason enum:
   ```rust
   fn classify_failure_reason(status: &str, error_msg: Option<&str>, job: &DbJob) -> FailureReason {
       match status {
           "timeout" => FailureReason::Timeout,
           "failed" => FailureReason::ExitNonzero,
           "error" => {
               match error_msg {
                   Some(msg) if msg.contains("network_target_unavailable") => {
                       FailureReason::NetworkTargetUnavailable
                   }
                   Some(msg) if msg.contains("image pull") || msg.contains("pull failed") => {
                       FailureReason::ImagePullFailed
                   }
                   Some(msg) if msg.contains("orphaned at restart") => {
                       FailureReason::Abandoned
                   }
                   _ => FailureReason::Unknown,
               }
           }
           _ => FailureReason::Unknown,
       }
   }
   ```
   Read the actual error_message strings used in `docker_preflight.rs`, `docker_pull.rs`, and `docker_orphan.rs` to ensure the string matching is correct. Adjust the contains() patterns to match the exact strings used in the codebase.
