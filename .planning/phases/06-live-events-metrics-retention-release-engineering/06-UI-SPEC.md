## Component Inventory (Phase 6 New/Modified)

### 1. Live Log Viewer (SSE mode)

Replaces the static log viewer on the Run Detail page when the run status is `running`.

**Container:**
```html
<div id="log-container">
  <!-- LIVE badge + heading -->
  <div class="flex items-center gap-2 mb-4">
    <h2 style="font-size:var(--cd-text-lg);font-weight:700">Log Output</h2>
    <span class="cd-badge cd-badge--running">LIVE</span>
  </div>

  <!-- SSE-connected log area -->
  <div id="log-lines"
       hx-ext="sse"
       sse-connect="/events/runs/{run_id}/logs"
       sse-swap="log_line"
       hx-swap="beforeend"
       style="background:var(--cd-bg-surface-sunken);padding:var(--cd-space-4);border-radius:8px;overflow-x:auto;max-height:600px;overflow-y:auto">
    <!-- log lines appended here by SSE -->
  </div>
</div>
```

**SSE event: `log_line`** -- each event appends one log line div matching the existing `log_viewer.html` format:
```html
<div class="{stderr_classes}" style="font-size:var(--cd-text-base);line-height:1.6">
  <span class="text-[var(--cd-text-secondary)] select-none" style="font-size:var(--cd-text-xs)">{stream}</span>
  {html_escaped_content}
</div>
```

**SSE event: `skip_marker`** -- inserted when broadcast::Lagged(n) is received:
```html
<div style="font-size:var(--cd-text-sm);color:var(--cd-status-disabled);padding:var(--cd-space-2) var(--cd-space-4);text-align:center;font-style:italic">
  [skipped {N} lines -- reload page for full log]
</div>
```

**SSE event: `run_complete`** -- triggers HTMX OOB swap replacing `#log-container` with the static paginated log viewer:
```html
<div id="log-container" hx-swap-oob="innerHTML">
  <!-- static log viewer from partials/log_viewer.html -->
</div>
```

**Scroll behavior:** Auto-scroll to bottom on each new `log_line` event. Use `scrollTop = scrollHeight` on the `#log-lines` container. Stop auto-scrolling if user manually scrolls up (detected by `scrollTop + clientHeight < scrollHeight - 50`).

**Max height:** `600px` with `overflow-y: auto`. Prevents the log viewer from pushing the page metadata card off-screen.

### 2. SSE Connection Error State

Displayed when the EventSource connection drops unexpectedly (not via `run_complete`):

```html
<div style="padding:var(--cd-space-4);text-align:center;color:var(--cd-status-disabled);font-size:var(--cd-text-base)">
  Unable to stream logs. <a href="" style="color:var(--cd-text-accent);text-decoration:underline">Refresh the page</a> to retry.
</div>
```

### 3. "Waiting for output" State

Displayed in the SSE log area before the first log line arrives:

```html
<div id="log-placeholder" style="padding:var(--cd-space-6);text-align:center;color:var(--cd-text-secondary);font-size:var(--cd-text-base)">
  Waiting for output...
</div>
```

Removed by the first `log_line` SSE event (use `hx-swap="innerHTML"` on first event, then switch to `beforeend`; or remove placeholder via JS on first event).

## Interaction Contracts

### SSE Log Streaming (UI-14)

| Interaction | Behavior |
|-------------|----------|
| User opens Run Detail for a running job | Page renders SSE log viewer with `LIVE` badge and "Waiting for output..." placeholder |
| First log line arrives via SSE | Placeholder removed, log line appended, auto-scroll active |
| Subsequent log lines arrive | Appended to bottom, auto-scroll if user has not scrolled up |
| User scrolls up in log viewer | Auto-scroll pauses (user reading history) |
| User scrolls back to bottom | Auto-scroll resumes |
| Slow subscriber falls behind | `[skipped N lines]` marker inserted in-stream |
| Run completes | `run_complete` SSE event fires, HTMX swaps in static paginated log viewer, `LIVE` badge disappears |
| SSE connection drops unexpectedly | Error message displayed with refresh link |
| User opens Run Detail for a completed run | Static paginated log viewer renders (no SSE connection attempted) |
| User opens Run Detail for a completed run with no logs | "No log output captured for this run." message |

### /metrics Endpoint (OPS-02)

| Interaction | Behavior |
|-------------|----------|
| Prometheus scrapes `GET /metrics` | Returns `text/plain; version=0.0.4; charset=utf-8` with all metrics |
| Browser hits `/metrics` | Returns same plain text (no HTML rendering) |

No UI changes needed -- this is a plain-text API endpoint.

### Retention Pruner (DB-08)

No UI interaction. Background task only. Pruning events logged via tracing (visible in Settings page uptime/status if the structured log mentions it, but no dedicated UI surface).

## Accessibility Notes

- SSE log viewer `#log-lines` container has `role="log"` and `aria-live="polite"` so screen readers announce new log lines without interrupting
- `LIVE` badge uses `aria-label="Live streaming"` for screen readers
- Skip marker and error states are visible to assistive technology (not hidden via `aria-hidden`)
- Auto-scroll does not prevent keyboard navigation within the log container
