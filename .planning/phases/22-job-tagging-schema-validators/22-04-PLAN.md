## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DbRunDetail.tags (Plan 03 output) → WebhookPayload.tags | `.clone()` is value-copy; no shared mutable state; tags already validated at config-load (Plan 02) before reaching DB write site (Plan 03) |
| WebhookPayload → wire JSON | `serde::Serialize` derive emits the field in declaration order; no operator-controlled key/value injection; tags are operator-supplied but charset-validated |
| webhook receiver → consumer | receiver decides what to do with tag values; cronduit's responsibility ends at delivery |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-04-01 | Tampering | tag value injection in payload | mitigate | tags pass through Plan 02's charset validator (`^[a-z0-9][a-z0-9_-]{0,30}$`) before reaching DB write; serde_json escapes any special chars in serialization (defense-in-depth — chars outside the validator's allowlist cannot reach this point) |
| T-22-04-02 | Information Disclosure | tag values delivered to webhook receivers | accept | tags are explicitly operator-supplied organizational metadata; the operator chose to attach these to the job AND configured the webhook URL — disclosure is intentional |
| T-22-04-03 | Denial of Service | unbounded tag count in payload | mitigate | Plan 02 D-08 cap of 16 tags per job pre-empts pathological payload sizes; payload size is bounded by 16 × 31 chars + JSON overhead = ~600 bytes maximum tags contribution |
| T-22-04-04 | Tampering | placeholder regression (until_p22 test still passes against vec![]) | mitigate | D-06.5 deletes the old test entirely; new test asserts NON-empty multi-tag round-trip; structural breadcrumb is gone after this plan |
| T-22-04-05 | Repudiation | webhook signature integrity preserved | accept | HMAC signing (Phase 19) wraps the entire payload including tags; tag changes already trigger signature changes — no new repudiation surface |
| T-22-04-06 | Elevation of Privilege | new external crate | accept | NO new external crates added (D-17); `serde_json` already used in payload module |
