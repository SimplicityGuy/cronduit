# Phase 13 — UI Design Contract (Observability Polish, rc.2)

> Visual and interaction contract for the three observability surfaces: dashboard sparkline + success-rate badge (OBS-03), job-detail duration p50/p95 card (OBS-04/OBS-05), and `/timeline` page (OBS-01/OBS-02). Generated by gsd-ui-researcher, verified by gsd-ui-checker. All decisions below are either (a) locked upstream in `13-CONTEXT.md` / `ROADMAP.md` / `REQUIREMENTS.md`, (b) carried forward from Cronduit's shipped `design/DESIGN_SYSTEM.md`, or (c) resolved by this spec for the Claude's-Discretion sub-areas called out in the phase prompt.

## Design System

| Property | Value |
|----------|-------|

| Preset | not applicable |

## Spacing Scale

All new surfaces use the existing `--cd-space-*` tokens from `DESIGN_SYSTEM.md` §4 (strict 4px grid). No new spacing tokens introduced.

| Token | Value | Phase 13 usage |
|-------|-------|----------------|
| `--cd-space-1` | 4px | Sparkline cell gap; timeline bar vertical padding; tooltip inner padding-y |
| `--cd-space-2` | 8px | Badge padding-x; Duration-card chip gap; tooltip inner padding-x |
| `--cd-space-3` | 12px | Pill-button padding-x; Duration-card heading margin-bottom |
| `--cd-space-4` | 16px | Timeline row vertical padding; Duration card inner row spacing |
| `--cd-space-6` | 24px | Duration card outer padding; timeline page top margin; pill-toggle margin-bottom |
| `--cd-space-8` | 32px | Timeline axis label bottom margin; empty-window block top/bottom padding |

**Exceptions:**
- Sparkline cell width is `6px` (multiple of 2, not 4) to accommodate 20 cells + 19 gaps inside the fixed `160px` column width. The cell **height** (`14px`) is an inline-flex measurement matched to `--cd-text-base` × `1.0` line-height and rounds to the nearest 2px; this is a sub-grid decorative element (same category as the existing `cd-badge` padding `2px 8px`).
- Timeline bar minimum width is `2px` so sub-second runs never vanish. Bars below this threshold (runs shorter than `window_total_ms * 2 / container_px`) render at the floor.
- Tooltip arrow (caret) is a `6px` triangle via CSS borders.
- Pulse animation uses `@keyframes` and is unaffected by spacing scale.

**Interactive behavior:**
- Hover over a cell: native `title` tooltip appears (browser default delay). No CSS hover.
- No `:focus-visible` ring on individual cells; the parent `<td>` cell is the tab stop (inherits from row hover).
- No click handler on cells (D-04).

**A11y:**
- Parent `<div class="cd-sparkline" role="img" aria-label="...">` announces the summary for screen readers.
- Individual `<span>` cells have no role (decorative within the img).
- `.cd-sparkline-badge` has a `title` for mouse and is discoverable via the SR summary.
- Color is not the sole signal: the `title` carries the status label in text.

**Responsive:** Parent `<div class="overflow-x-auto">` (already wrapping the dashboard table at `dashboard.html:39`) keeps the table scrollable horizontally at narrow widths. The sparkline column has `min-width: 180px` (= 20 × 6px + 19 × 4px + badge 36px + padding 40px) so it never collapses. At `< 640px` width, horizontal scroll activates — no separate mobile layout.

**"Now" vertical indicator (Claude's Discretion resolution):** DEFERRED. Implementing a current-time vertical dashed line that spans all rows requires an extra positioned overlay that complicates the tooltip stacking context and (for 7d windows) shifts visibly every 30s on poll, creating layout jitter. The pulsing running bar already signals "this is live". If a future operator asks for it, revisit in v1.2.

**Touch / narrow-width behavior:**
- Parent `.cd-timeline` has `overflow-x: auto` and `min-width: 640px`. Below 640px viewport, the page scrolls horizontally.
- On touch devices, CSS `:hover` tooltip does not appear; the bar `title` attribute shows on long-press (D-09 fallback). Screen readers announce the `title` on focus.
- Tap target: bars shorter than `min-width: 2px` inflate to 2px; bars shorter than WCAG 44px touch target do NOT inflate (we accept this — operators on mobile do secondary navigation via dashboard, not timeline; timeline is a primary-UI surface). This is a known acceptable limitation.

**A11y:**
- Each bar is an `<a>` with an `href`, so it's keyboard-navigable (Tab order: left→right, row-by-row, top-to-bottom within the timeline; alphabetical by job name since rows are ordered alphabetically).
- `:focus-visible` ring (green, matches project convention) is always present.
- `title` attribute is present on every bar for screen-reader announcement.
- `aria-current="page"` on the active pill.
- `role="tooltip"` on `.cd-tooltip` (advisory — since we hide it by default it isn't part of the accessibility tree until hover, at which point the `title` has already announced).
- `prefers-reduced-motion: reduce` kills the pulse.

**Polling:**
- `hx-trigger="every 30s"` on `#timeline-body`.
- `hx-include="[name='window']"` preserves the current window selection across the poll.
- `hx-swap="outerHTML"` replaces the whole body (pill toggle outside the swap target, so it does NOT flicker).
- NO `htmx.load` modifier to pause-when-hidden — kept simple; operator's browser handles background tab throttling natively.
