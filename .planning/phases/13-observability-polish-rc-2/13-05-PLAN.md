    **Handler body:**
    ```
    pub async fn timeline(
        State(state): State<AppState>,
        Query(params): Query<TimelineParams>,
    ) -> impl IntoResponse {
        // Window selection — default 24h; strict allow-list
        let window = match params.window.as_deref() {
            Some("7d") => "7d",
            _ => "24h",
        };

        let (window_duration, tick_step, tick_count, tick_format_24h, daily_ticks): (
            ChronoDuration, ChronoDuration, usize, bool, bool
        ) = match window {
            "7d" => (ChronoDuration::days(7), ChronoDuration::days(1), 7, false, true),
            _ => (ChronoDuration::hours(24), ChronoDuration::hours(2), 12, true, false),
        };

        let now_utc: DateTime<Utc> = Utc::now();
        let window_start_utc: DateTime<Utc> = now_utc - window_duration;
        let total_seconds = window_duration.num_seconds() as f64;
        let tz: Tz = state.tz;

        // Fetch
        let runs = queries::get_timeline_runs(&state.pool, &window_start_utc.to_rfc3339())
            .await
            .unwrap_or_default();

        let truncated = runs.len() == 10_000;

        // Group by job, preserving alphabetical order (query's ORDER BY j.name ASC
        // already sorts; BTreeMap gives deterministic grouping).
        let mut by_job: BTreeMap<String, TimelineJobRow> = BTreeMap::new();
        for run in runs {
            let row = by_job.entry(run.job_name.clone()).or_insert_with(|| TimelineJobRow {
                id: run.job_id,
                name: run.job_name.clone(),
                bars: Vec::new(),
            });

            // Parse start_time — try RFC3339 first, fallback to "YYYY-MM-DD HH:MM:SS"
            let start_utc = parse_db_timestamp(&run.start_time).unwrap_or(now_utc);
            // Clamp start to window_start (runs that started before the window but
            // survived the SQL filter should not exist given start_time >= window_start,
            // but guard anyway).
            let start_clamped = start_utc.max(window_start_utc);

            // For terminal runs use end_time; for running runs extend to now_utc.
            let end_utc = match run.end_time.as_deref() {
                Some(s) => parse_db_timestamp(s).unwrap_or(now_utc),
                None => now_utc,
            };
            // Clamp end to now_utc (never render into the future).
            let end_clamped = end_utc.min(now_utc);

            let left_secs = (start_clamped - window_start_utc).num_seconds() as f64;
            let width_secs = (end_clamped - start_clamped).num_seconds() as f64;

            let left_pct = (left_secs / total_seconds * 100.0).clamp(0.0, 100.0);
            let raw_width_pct = width_secs / total_seconds * 100.0;
            // Never extend past the right edge; minimum width handled in CSS via var(--cd-timeline-bar-min-width).
            let width_pct = raw_width_pct.clamp(0.0, 100.0 - left_pct);

            let status_lower = run.status.to_lowercase();
            let duration_display = format_duration_ms_floor_seconds(run.duration_ms);

            let start_local = start_utc.with_timezone(&tz);
            let end_local = end_utc.with_timezone(&tz);

            row.bars.push(TimelineBar {
                run_id: run.run_id,
                job_id: run.job_id,
                job_run_number: run.job_run_number,
                status_upper: status_lower.to_uppercase(),
                status: status_lower,
                // Format to 3 decimals in Rust so the rendered CSS is
                // `left:12.500%` not `left:12.499999999999998%`. askama will
                // interpolate this String verbatim.
                left_pct: format!("{:.3}", left_pct),
                width_pct: format!("{:.3}", width_pct),
                duration_display,
                start_time_str: start_local.format("%H:%M:%S").to_string(),
                end_time_str: if run.end_time.is_some() {
                    end_local.format("%H:%M:%S").to_string()
                } else {
                    "now".to_string()
                },
            });
        }

        let jobs: Vec<TimelineJobRow> = by_job.into_values().collect();

        // Axis ticks
        let mut axis_ticks: Vec<TimelineAxisTick> = Vec::with_capacity(tick_count);
        for i in 0..tick_count {
            let tick_utc = window_start_utc + tick_step * (i as i32);
            let tick_local = tick_utc.with_timezone(&tz);
            let tick_secs = (tick_utc - window_start_utc).num_seconds() as f64;
            let left_pct = (tick_secs / total_seconds) * 100.0;
            let label = if daily_ticks {
                tick_local.format("%a").to_string()  // Mon, Tue, ...
            } else if tick_format_24h {
                tick_local.format("%H:00").to_string()
            } else {
                tick_local.format("%H:%M").to_string()
            };
            axis_ticks.push(TimelineAxisTick {
                left_pct: format!("{:.3}", left_pct),
                label,
            });
        }

        TimelinePage {
            window: window.to_string(),
            jobs,
            axis_ticks,
            truncated,
        }
        .into_web_template()
        .into_response()
    }

    /// Parse a DB timestamp stored as either RFC3339 or "YYYY-MM-DD HH:MM:SS".
    /// Mirrors the dashboard handler's fallback idiom.
    fn parse_db_timestamp(s: &str) -> Option<DateTime<Utc>> {
        if let Ok(dt) = DateTime::parse_from_rfc3339(s) {
            return Some(dt.with_timezone(&Utc));
        }
        if let Ok(ndt) = chrono::NaiveDateTime::parse_from_str(s, "%Y-%m-%d %H:%M:%S") {
            return Some(ndt.and_utc());
        }
        None
    }
    ```

    **Bar ordering within each job's row** is already alphabetical-by-name at the job level (BTreeMap key) and chronological within the row (the SQL query orders by `start_time ASC` — the `row.bars.push` preserves that order).

    **Clippy note:** the tick-loop uses `tick_step * (i as i32)` with `ChronoDuration` — chrono's Duration `Mul` accepts `i32`. If clippy complains, use `tick_step * i32::try_from(i).unwrap_or(0)`.

    **DST handling note (Research Risk #7):** the approach above formats each tick's UTC into the local tz via `.with_timezone(&tz).format(...)`. DST transitions are handled by `chrono_tz` — no manual offset math. If a 7d window crosses spring-forward, the 03:00 hour that doesn't exist for that day would still render its tick at the computed UTC position, but the label would show the next valid local time. Acceptable for v1.1.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP → timeline handler | `GET /timeline` read-only; query param `window` strictly validated to `{24h, 7d}` with default fallback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-05-01 | Tampering | XSS via job_name rendered into HTML + tooltip | mitigate | askama auto-escapes `{{ }}`. `job_name` originates from operator-authored `cronduit.toml`, and the `title="{{ job.name }}"` attribute on `.cd-timeline-row-label` is auto-escaped. No raw interpolation. |
| T-13-05-02 | Tampering | XSS via `bar.status` rendered into class name | mitigate | View-model pre-lowers status via `.to_lowercase()`; status values come from DB enum column with constrained domain. `{{ bar.status }}` interpolation into class is still safe because the class attribute is quoted and any unexpected char would only break the class name, not escape the attribute. |
| T-13-05-03 | Info disclosure | Timeline enumerates all enabled jobs' run history | accept | Dashboard already exposes the same data. v1 web UI is unauthenticated (THREAT_MODEL.md); loopback-bind default stands. Plan adds no new boundary. |
| T-13-05-04 | DoS | Attacker triggering 30s HTMX poll at high frequency | accept | Poll rate is HTMX-side (`hx-trigger="every 30s"`); no server-side timer. Per-request cost is bounded by `LIMIT 10000` and `idx_job_runs_start_time`. Homelab scale → negligible. |
| T-13-05-05 | DoS | Query returning 10000 rows → renders 10k `<a>` elements | accept | Truncation banner informs operator; browser can render 10k elements; homelab scale almost never reaches this. |
| T-13-05-06 | Tampering | `window` param value smuggling (e.g., `?window=drop%20table`) | mitigate | Handler's strict allow-list `match params.window.as_deref() { Some("7d") => "7d", _ => "24h" }` ignores any unexpected value. No value passed to SQL. |
| T-13-05-07 | Open redirect via bar href | n/a | — | Every bar's `href="/jobs/{job_id}/runs/{run_id}"` is a relative URL with integers interpolated; no user-controlled URL component. |
</threat_model>
