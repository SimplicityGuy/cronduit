```rust
//! Per-run control plane: cancel token + stop reason (SCHED-10, D-09).

use std::sync::Arc;
use std::sync::atomic::{AtomicU8, Ordering};
use tokio_util::sync::CancellationToken;

#[repr(u8)]
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum StopReason {
    Shutdown = 0,
    Operator = 1,
}

impl StopReason {
    pub fn from_u8(v: u8) -> Self {
        match v {
            1 => Self::Operator,
            _ => Self::Shutdown,
        }
    }
}

#[derive(Clone)]
pub struct RunControl {
    pub cancel: CancellationToken,
    pub stop_reason: Arc<AtomicU8>,
}

impl RunControl {
    pub fn new(cancel: CancellationToken) -> Self {
        Self {
            cancel,
            stop_reason: Arc::new(AtomicU8::new(StopReason::Shutdown as u8)),
        }
    }

    /// Set the reason atomically BEFORE firing cancel.
    /// The executor reads the atomic AFTER cancel.cancelled() yields,
    /// which happens-after the store by SeqCst ordering.
    pub fn stop(&self, reason: StopReason) {
        self.stop_reason.store(reason as u8, Ordering::SeqCst);
        self.cancel.cancel();
    }

    pub fn reason(&self) -> StopReason {
        StopReason::from_u8(self.stop_reason.load(Ordering::SeqCst))
    }
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scheduler cancel token → executor cancel arm | `RunControl::stop(Operator)` stores `stop_reason=1` with SeqCst ordering BEFORE firing the cancel token. The executor's cancel arm reads the atomic AFTER `cancel.cancelled()` yields. Atomic ordering is the ONLY mechanism guaranteeing the executor sees the correct reason. |
| Executor process-group kill boundary | `.process_group(0)` spawn + `libc::kill(-pid, SIGKILL)` kills the process group atomically. Shell-pipeline grandchildren are reaped through the pgid. D-17 locks this pattern. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Tampering (memory ordering) | `RunControl::stop` atomic ordering | mitigate | `stop_reason.store(..., Ordering::SeqCst)` issued BEFORE `cancel.cancel()`. The `Rust atomics` memory model guarantees happens-before semantics for SeqCst ordered stores, so the executor's `control.reason()` load (also SeqCst) observes the operator reason. Tests `stop_reason_operator_roundtrip` and `shutdown_cancel_stays_shutdown` exercise both paths. |
| T-10-03-02 | Tampering (process lifecycle) | Process-group kill regression | mitigate | D-17 preservation lock: `.process_group(0)` spawn + `libc::kill(-pid_i32, SIGKILL)` must NOT be replaced with `kill_on_drop(true)` (would orphan shell-pipeline grandchildren per PITFALLS §1.3 / Research Correction #1). Acceptance criteria grep for `process_group(0)` and `! grep kill_on_drop`. Full integration test for grandchildren reaping lives in plan 10-10 (T-V11-STOP-07..08). |
| T-10-03-03 | Denial of Service (executor wedge) | Kill path blocks forever | mitigate | After `kill_process_group`, the code awaits `child.wait()` and stdout/stderr task joins. SIGKILL is non-maskable on Linux, so `child.wait()` always returns. No wait-with-timeout needed. |
| T-10-03-04 | Information Disclosure | error_message leakage | accept | error_message literal strings are "stopped by operator" and "cancelled due to shutdown". No user input, no stack trace, no secrets. |

ASVS L1 coverage: V5 Input Validation (atomic-ordered state machine rather than shared mutable state), V7 Error Handling (no information leakage in cancel-arm error messages).
