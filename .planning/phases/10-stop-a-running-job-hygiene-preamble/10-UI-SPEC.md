# Phase 10 — UI Design Contract

> Visual and interaction contract for the Stop-a-Running-Job button and the new `stopped` terminal status. Anchored to the Cronduit terminal-green design system (`design/DESIGN_SYSTEM.md`); no new visual language is invented. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-authored CSS + Askama templates; project is Rust/Askama/Tailwind/HTMX — NOT React/Vite) |
| Preset | not applicable (shadcn gate skipped — stack is server-rendered) |
| Component library | none (hand-rolled `cd-*` token system in `assets/static/app.css` + `design/DESIGN_SYSTEM.md`) |
| Icon library | none (pure CSS/text buttons; optional inline SVG square/stop glyph per D-Discretion — planner choice) |
| Font | `JetBrains Mono` 400/500/700 — monospace-everything brand lock; never mix sans/serif |

**Source of truth:** `design/DESIGN_SYSTEM.md` + `assets/static/app.css` (`:root` variable block). Phase 10 extends, never replaces.

## Scope of This Contract

Phase 10 touches exactly these surfaces:

1. **`templates/pages/run_detail.html`** — new Stop button in the empty right-side action slot of the `Run #N` header row (L15–18), gated by `is_running`. Status badge at L26 gains `stopped` rendering.
2. **`templates/partials/run_history.html`** — new per-row compact "Stop" text button for rows where `status = 'running'`. Status badge cell (L33) gains `stopped` rendering.
3. **`assets/static/app.css`** — new `--cd-status-stopped` + `--cd-status-stopped-bg` tokens (dark + light), new `.cd-badge--stopped` modifier, new `.cd-btn-stop` button class.
4. **`design/DESIGN_SYSTEM.md`** — `stopped` row appended to Status Colors table (§2.2) + Status Background Tints table (§2.2) in the same commit so the design doc stays source-of-truth.
5. **Dashboard / index** — anywhere a status badge renders must pick up `cd-badge--stopped` via the shared template macro. No dashboard-specific visual changes beyond this cascade.

Out of scope for this UI-SPEC: login/auth, keyboard shortcuts, icon glyph selection (planner discretion), hex values for tokens outside the stopped family.

### Why slate-400 / slate-500 for `stopped`

- **Neutral, not red.** D-02 locks `stopped ≠ failure`. Reusing `--cd-status-error` would double-count stops into the failure denominator visually (Phase 13 success-rate sparkline explicitly excludes `stopped` per OBS-03).
- **Not yellow either.** `--cd-status-disabled` is already the "paused by config" affordance that Phase 14 bulk-disable will use. Operator-interrupt is a different semantic — recent terminal state, not ongoing config state.
- **Matches peer tools.** GitHub Actions, Buildkite, and Jenkins all render cancelled/stopped in a neutral gray. Operators transferring from those tools read gray as "stopped by me" without training.
- **Brand harmony.** Slate-400/500 sits in the same ~55–65% lightness band as the other four status colors (per DESIGN_SYSTEM.md §2.2 lock) and is tonally cool enough that it does not fight the terminal-green accent.
- **Contrast lock.** Slate-400 `#94a3b8` on `--cd-bg-surface` `#0a0d0b` yields contrast ratio ≈ 8.9:1 (passes WCAG AAA body). Slate-500 `#64748b` on `#ffffff` yields ≈ 5.4:1 (passes WCAG AA body + AA large text). Badge text is xs/bold on tinted background — contrast of slate-400 on its own 12%-alpha tint against the surface remains ≈ 8.5:1.

### Destructive color policy

**There is no destructive color used for the Stop action in Phase 10.** This is an explicit D-05 decision. Operator-stop is not a failure and is not alarming — it is a deliberate, reversible-in-intent command. The Stop button is a **neutral outline button** that tints toward `--cd-status-stopped` on hover. `--cd-status-error` is reserved for run failure badges and error message cards only.

## Copywriting Contract

| Element | Copy | Rationale |
|---------|------|-----------|
| Primary CTA (run_detail page-action slot) | `Stop` | One word, imperative verb. Matches the terse terminal-tool voice from DESIGN_SYSTEM.md §1.2. Symmetric with existing `Run Now` button. |
| Compact CTA (run_history row) | `Stop` | Same word across both surfaces so operators build one muscle memory. No `Stop run` / `Kill` / `Terminate` synonyms — single canonical verb. |
| Button `aria-label` | `Stop run #{N}` (where N is the global `job_runs.id`) | Screen-reader hint that disambiguates which run is being stopped when multiple Stop buttons are on one page (run_history). On run_detail the `#N` is redundant with the page title but included for consistency. |
| Success toast (normal path) | `Stopped: {job_name}` | Symmetric with existing `Run queued: {job_name}` toast (api.rs:65). Past-tense affirms the action completed. Level: `info` (matches Run Now; not a warning). |
| Toast level | `info` | `HxEvent::new_with_data("showToast", json!({"message": "...", "level": "info"}))` — verbatim pattern from Run Now. |
| Race-case feedback | **No toast. `HX-Refresh: true` only.** | D-07: silence-is-success. The refreshed page shows the real natural terminal status (`success` / `failed` / `timeout`) and that IS the message. Never show "Run already finished" — it creates noise for a condition that isn't an error. |
| Error state (channel closed) | HTTP 503 `Service Unavailable` + body `Scheduler is shutting down` | Verbatim from Run Now. Handler returns 503; no HTMX toast. The browser-native error behavior is correct here because the whole app is going down. |
| New badge label | `STOPPED` (uppercase, letter-spacing 0.1em; rendered by template via `run.status_label`) | Matches existing badge text convention: `SUCCESS`, `RUNNING`, `FAILED`, `TIMEOUT`, `DISABLED`. Eight characters — fits in all existing table cells without reflow. |
| Status in metadata tooltip / secondary hint | `stopped` (lowercase, canonical DB string) | Matches DB-side canonical string per D-10. Used in JSON APIs, `cronduit_runs_total{status="stopped"}` metric label, and log messages. |
| Destructive confirmation dialog | **None — toast-only.** | D-07 + project policy: no confirmation dialog for Stop, consistent with Run Now having none. `THREAT_MODEL.md` gets a one-line note (per roadmap Phase 14 design) that Stop widens blast radius for anyone with UI access. No design work on a dialog — the dialog does not exist. |
| Empty state (no running runs to stop) | N/A — Stop button is not rendered when `is_running == false`. No placeholder text. | The absence of the button is the message. Operators cannot be confused about a missing affordance because there is nothing to stop. |
| Log viewer transition | (unchanged) | Phase 10 does NOT modify the `Log Output` / `LIVE` badge / `Waiting for output…` placeholder. Those belong to Phase 11. |

### Copy not used

The following phrases are **explicitly rejected** and must not appear in any template:
- `Cancel`, `Kill`, `Terminate`, `Abort`, `End`, `Halt` — `Stop` is canonical.
- `Are you sure?`, `This cannot be undone`, `Confirm stop` — no confirmation dialog exists.
- `Run stopped successfully` / `Successfully stopped run` — verbose, non-symmetric with Run Now.
- `Too late — this run already finished` — race case is silent; no message.

## Interaction States (exhaustive)

For the Stop button on both surfaces:

| State | Trigger | Visual | Behavior |
|-------|---------|--------|----------|
| **Hidden** | `is_running == false` (run is already in a terminal state) | Button element is not rendered at all. | Absence of affordance = "nothing to stop." No placeholder, no disabled ghost. |
| **Idle** | `is_running == true`, no pointer | Transparent background, 1px `--cd-border`, `--cd-text-primary` text. | Clickable. |
| **Hover** | Pointer over button | Background → `--cd-status-stopped-bg`; border → `--cd-status-stopped`; text → `--cd-status-stopped`. 150ms ease transition. | Tooltip-equivalent: hover tint signals "this will stop the run." |
| **Focus (keyboard)** | Tab to button | Idle visuals + `--cd-border-focus` border + `0 0 0 2px var(--cd-green-dim)` ring. | Matches every other button's focus pattern. |
| **Active (pressed)** | `mousedown` / `Enter` / `Space` | Hover visuals + `transform: translateY(1px)` (subtle press-down cue). | |
| **Loading** | Form submission in flight (HTMX `hx-indicator` / `aria-busy`) | Hover visuals, `opacity: 0.6`, `cursor: not-allowed`, `pointer-events: none`. Button text remains `Stop` (no spinner per project style — HTMX handles re-render). | Prevents double-submit. Released when the page reloads from `HX-Refresh`. |
| **Disabled** | Not applicable in v1.1 (no partial-disabled state) | (n/a) | Button is either hidden (not running) or interactive (running). |
| **Error (channel closed)** | 503 response | No UI element reflects this — the browser receives 503 from the form post. Operator sees the HTTP error text. | Deliberate: the scheduler is going away, so normal UX is moot. |
| **Success** | 200 OK + `HX-Refresh: true` | Page reloads immediately. Toast `Stopped: {job_name}` appears in top-right toast slot for 4s (existing toast duration). New badge renders as `STOPPED` (slate). | |
| **Race-win-by-natural** | 200 OK + `HX-Refresh: true`, no toast | Page reloads. Badge renders the natural terminal status (success/failed/timeout). No message. | D-07 locked: silence is success. |

### Confirmation pattern for the destructive action

**No confirmation dialog.** This is the explicit phase decision (D-07) and is consistent with `Run Now` having none. The affordance model is:

1. **Visual restraint** — neutral outline button (not alarming red) tells operators this is a precise, reversible-in-intent command, not a fleet-wide nuke.
2. **Post-action recoverability** — a stopped run is just a terminal status; the next scheduled firing runs normally. No data is destroyed.
3. **Toast feedback** — the success toast confirms the action occurred. No modal interstitial.
4. **Threat model acceptance** — v1.x posture is trusted-LAN / reverse-proxy. Stop widens blast radius, documented as a `THREAT_MODEL.md` one-line note (no design work). Future auth (v2) will gate the button, not wrap it in a dialog.

A confirmation dialog would:
- Break symmetry with `Run Now` (also mutative, also no dialog)
- Add 2–3 clicks to a command operators will use for stuck jobs under time pressure
- Create an accessibility burden (focus trap, escape handling, aria-modal) for zero safety benefit given items 1–3

## HTMX Interaction Contract

Exact wire format — planner and executor must not reinvent.

**Form element (both surfaces):**

```html
<form hx-post="/api/runs/{{ run.id }}/stop"
      hx-swap="none"
      hx-disabled-elt="this"
      style="display:inline">
  <input type="hidden" name="csrf_token" value="{{ csrf_token }}">
  <button type="submit"
          class="cd-btn-stop{% if compact %} cd-btn-stop--compact{% endif %}"
          aria-label="Stop run #{{ run.id }}">
    Stop
  </button>
</form>
```

**Server response (success path):**

- HTTP 200
- Header `HX-Trigger: {"showToast":{"message":"Stopped: JOB_NAME","level":"info"}}`
- Header `HX-Refresh: true`
- Body empty

**Server response (race — run already finalized):**

- HTTP 200
- Header `HX-Refresh: true`
- **No** `HX-Trigger` header (D-07)
- Body empty

**Server response (scheduler shutting down):**

- HTTP 503
- Body `Scheduler is shutting down`

**Server response (CSRF mismatch):**

- HTTP 403
- Body `CSRF token mismatch`

**Server response (run not found / not running at all):**

- HTTP 404 (run never existed) OR silent 200 + `HX-Refresh: true` (run exists but already terminal — race case path). Planner picks in PLAN.md per D-07 Claude's Discretion note.

## Accessibility Contract

- **Keyboard:** Stop button is a `<button type="submit">` inside a form — reachable via Tab, activatable via Enter and Space (browser default). No custom key handler.
- **Focus visible:** green focus glow (`var(--cd-green-dim)`) matches every other button in the app. Never remove `:focus-visible` outline without a replacement.
- **ARIA:** `aria-label="Stop run #{{ run.id }}"` on the button; no `role` override (button is already a button).
- **Loading state:** `hx-disabled-elt="this"` sets `disabled` on the submit button during the request. For screen-reader announcement of the state change, rely on the post-refresh page title/status badge — the toast is an `HX-Trigger` event that the existing toast handler announces via `aria-live="polite"` region (existing v1.0 infrastructure; no new work).
- **Contrast:** stopped color tokens pass WCAG AA (dark mode AAA 8.9:1, light mode AA 5.4:1) — verified above in § Color.
- **Reduced motion:** `transition: background .15s ease` is within reduced-motion tolerance. If `@media (prefers-reduced-motion: reduce)` is ever added, the transition should be removed without changing functionality. Not blocking for Phase 10.
