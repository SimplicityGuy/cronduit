## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config-validate → bin-wire-up | Validated `WebhookConfig` flows from cfg.jobs into the per-job map; map is the only state the dispatcher consults |
| test isolation | Each integration test starts its own MockServer + DbPool; counters are delta-asserted (not absolute) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-26 | Tampering | Webhook config bypasses validators by sneaking through bin layer | mitigate | Bin layer reads from `cfg.jobs[i].webhook` AFTER `apply_defaults` AND AFTER `run_all_checks` (existing flow at parse_and_validate); no re-validation needed but no bypass either |
| T-18-27 | Repudiation | Test asserts on absolute counter values (flake) | mitigate | Use baseline + delta pattern from `tests/v12_webhook_queue_drop.rs:71-78` — every counter test reads baseline before action and asserts `final - baseline == expected_delta` |
| T-18-28 | Tampering | wiremock test passes locally but fails in CI due to port collision | mitigate | `MockServer::start().await` binds to ephemeral port; `server.uri()` returns the actual URL; no hardcoded ports |
| T-18-29 | Spoofing | Integration test verifies signature with wrong secret | mitigate | Tests construct `WebhookConfig.secret = SecretString::from("known-secret")`; signature is recomputed inside the test using the same secret + the body bytes captured from `received.body` |
| T-18-30 | Tampering | Body-byte drift between sign and send invalidated by HTTP framing | accept | Pitfall B mitigation lives in HttpDispatcher (Plan 04); this plan's integration tests verify the receiver-side outcome — if drift existed, signature verification on `received.body` would fail |
</threat_model>
