## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cronduit → checked-in fixture file | The fixture under `tests/fixtures/webhook-v1/` is committed plaintext; an attacker who lands a malicious commit could swap the test secret/payload to mask a sign-side regression. Mitigated by: (a) PR-only workflow per D-18, (b) the Rust test re-derives `payload.json` from `WebhookPayload::build` so a tampered payload would mismatch the re-derivation, (c) `expected-signature.txt` is also re-derived from `sign_v1` so swapping it requires also breaking the existing in-module HMAC pipeline. |
| Fixture secret bytes ↔ HMAC computation | The secret is plaintext-checked-in (not a real secret) — clearly labeled "test-fixture-secret-not-real" with a comment header in `secret.txt` and a loud warning in the fixture README. |
