<objective>
Migrate the webhook metrics surface from P18's two flat counters (`_sent_total`, `_failed_total`) to the labeled family `cronduit_webhook_deliveries_total{job, status}` plus the new per-attempt histogram `_delivery_duration_seconds{job}` plus the `_queue_depth` gauge — all eagerly described + zero-baselined at boot per the existing `src/telemetry.rs` discipline.

Purpose: Lock Success Criterion 4: "An operator scraping `/metrics` sees the new `cronduit_webhook_*` family eagerly described at boot." This is the operator-visible API contract for v1.2 webhooks; once rc.1 ships, dashboards key off these metric names. The P18 flat counters are removed (breaking change documented in Plan 07's docs); the P15 channel-saturation counter is preserved (non-breaking).
Output: Modified `src/telemetry.rs` (3 describes + closed-enum status seed + histogram bucket config) + modified `src/webhooks/dispatcher.rs` (success arm + histogram + remove flat counter calls) + modified `src/webhooks/retry.rs` (terminal-failure increments) + new integration test `tests/v12_webhook_metrics_family.rs` + updated 3 existing P18 webhook metric tests.
</objective>
