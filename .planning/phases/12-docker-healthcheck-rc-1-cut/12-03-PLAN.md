<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dockerfile HEALTHCHECK CMD → docker daemon → containerized binary exec | The Docker daemon executes the CMD as the container user; exec-form CMD bypasses shell expansion. |
| HEALTHCHECK CMD argv → `/cronduit health` exit code | Exit code IS the health signal; container marked unhealthy only on `Ok(1)` from Plan 02. |
| Container `(unhealthy)` state → orchestrator (compose / k8s) restart policy | Operators who configure restart-on-unhealthy depend on this signal being correct. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-03-01 | Tampering | HEALTHCHECK CMD shell injection | mitigate | Use exec form `CMD ["/cronduit", "health"]` — no shell expansion, no argv interpolation. Acceptance criterion grep-checks the exact bracketed form. |
| T-12-03-02 | Denial of Service | Migration backfill on first-boot exceeds default start-period | mitigate | `--start-period=60s` chosen per Phase 11 D-12 cross-reference (HTTP listener binds after backfill); D-12 of Phase 11 also added INFO progress logging during backfill so operators see progress. |
| T-12-03-03 | Information Disclosure | Healthy-state masking after a hung scheduler | accept | Phase 12 only consumes the existing `/health` endpoint (which returns `degraded` if DB is unreachable per `src/web/handlers/health.rs`). Strengthening the JSON contract is out of scope; deferred per CONTEXT.md (no `/healthz` "starting" state). |
| T-12-03-04 | Elevation of Privilege | HEALTHCHECK runs as root | mitigate | HEALTHCHECK is placed AFTER `USER cronduit:cronduit` (L127); the probe runs as the unprivileged container user, not root. Acceptance criterion verifies USER < HEALTHCHECK < ENTRYPOINT ordering. |
| T-12-03-05 | Tampering | Dockerfile drift across multi-arch builds | mitigate | The `Dockerfile` is single-source for both amd64 and arm64 (per CLAUDE.md "Multi-arch via cargo-zigbuild"); HEALTHCHECK is architecture-independent (it's a metadata directive, not a compile target). Plan 04's compose-smoke workflow exercises amd64; release.yml's existing matrix exercises both arches at tag-push time. |
